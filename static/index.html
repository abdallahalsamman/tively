How it works

Start and end with the same software you use to invoice the outsourcing companies anywyas: the spreadsheet!

Create requirements

Start off by creating requirements on a spreadsheet. Each row in a spreadsheet will link to an issue in the code management tool used by the outsourcing company such as Github, Gitlab or Bitbucket

Link requirement with issue tracker

You can jump immediately from the spreadsheet to the corresponding issue in the issue tracker the developers will be using, and see/edit the detailed requirements yourself

See changed files

You can trace the files and exact lines of code changes back to the issue in the issue tracker (which in turn is traced to business requirement). You may not understand the changes, but any tech person you hire can. Keep the developers honest!

Track hours worked

Engineers will add how many hours they spent on each issue on the issue tracker, and that will be immediately visible on the spreadshet

Make high level Analysis

You can view how much percent of the project has been done in specific categories, measure that against the estimated time given by the consulting company, estimate how much work is left given the current development speed and much more
